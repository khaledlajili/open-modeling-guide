# Open Modeling Guide

**Think in models. Build with confidence.**

[![Community Driven](https://img.shields.io/badge/Community-Driven-blue)](https://github.com/khaledlajili/open-modeling-guide)
[![Open Source](https://img.shields.io/badge/Open-Source-green)](LICENSE)
[![Open Source](https://img.shields.io/badge/Open-Source-green)](LICENSE) [![License: CC BY-SA](https://img.shields.io/badge/License-CC%20BY--SA-lightgrey)](https://creativecommons.org/licenses/by-sa/4.0/)

We believe that modeling is a fundamental skill in modern system design — essential for understanding and building complex software systems.

In practice, modeling standards and specifications are often difficult to approach. They are fragmented, highly formal, and rarely explained in a way that connects directly to real engineering work.

As a result, many engineers and learners struggle to move from abstract definitions to practical understanding.

We believe that this gap should not exist.

We envision a space where modeling concepts, standards, and system design ideas are explained clearly, structured logically, and made accessible to anyone who wants to understand how systems are designed.

---

Open Modeling Guide is a structured, open-source knowledge base that focuses on making modeling concepts understandable and usable.

It does not aim to replace official specifications. Instead, it helps bridge the gap between formal modeling standards and practical system understanding.

The first set of guides begins with Model-Driven Architecture (MDA) as a starting point to explore this space of ideas.

## Running the Guide Locally

This guide is built with [Docusaurus](https://docusaurus.io/), a modern static website generator. To run it locally for development or exploration:

```bash
# 1. Clone and navigate
git clone https://github.com/khaledlajili/open-modeling-guide.git
cd open-modeling-guide

# 2. Install dependencies
npm install  # or yarn install

# 3. Start the development server
npm start    # or yarn start
```
Once running, open http://localhost:3000 in your browser. The site will automatically reload as you make changes.
