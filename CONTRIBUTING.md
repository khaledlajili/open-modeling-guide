# Contributing to Open Modeling Guide

Thank you for contributing.

Open Modeling Guide is a simple, open-source effort to make modeling concepts and standards easier to understand and more practical to use.

We welcome improvements of all kinds — especially those that improve clarity.

---

## What you can contribute

You can help by:

- Improving explanations (clarity, simplicity, structure)
- Fixing mistakes or unclear parts
- Adding examples or diagrams
- Suggesting better ways to explain a concept
- Expanding existing guides (MDA, UML, BPMN, etc.)
- Reporting issues or confusing sections

---

## How to start

1. Fork the repository
2. Create a new branch
3. Make small and focused changes
4. Keep explanations clear and simple
5. Submit a pull request

---

## Writing style

To keep the guide consistent:

- Prefer simple and direct language
- Explain ideas step by step
- Avoid unnecessary jargon
- Focus on understanding, not theory
- Use examples when possible

---

## Pull Requests

When submitting a PR:

- Keep changes small and focused
- Explain what you improved and why
- Link to the related issue (if any)

We review contributions as quickly as possible.

---

## Issues

If something is unclear, outdated, or missing:

Open an issue and describe:

- what is confusing
- where it happens
- what you suggest instead (if possible)

---

## Code of Conduct

Be respectful and constructive.

This is a learning-focused project, and all contributions should help improve understanding.

---

Thank you for helping improve Open Modeling Guide.
